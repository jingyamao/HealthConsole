# 使用 Agent / AI 驱动构建的具体成果

本项目围绕医疗病历管理引入 AI/Agent 能力，解决四个核心痛点：(1) 病历查阅低效——自然语言查询 Agent 支持医生直接提问，自动转换为 Prisma 查询并返回格式化答案；(2) 数据分析依赖人工——AI 综合评估将患者 9 类病历格式化为上下文，调用 LLM 输出 7 维度结构化结果；(3) 知识检索脱节——17 篇医学文档经嵌入向量化后，对话时自动余弦检索，结果注入 LLM 实现 RAG 循证问答；(4) 表单填写繁琐——提供 tags/kvlist/rows 三种可视化组件替代 JSON 文本框。

架构上采用单 Agent 多工具编排（非多 Agent 协作），AgentManager 执行「意图识别 → 工具调度 → 结果持久化」链路，可调度 8 个工具。核心含两条长推理链：databaseQuery 三步串行 LLM（生成查询计划 → 确定性 Prisma 执行 → 自然语言格式化），RAG 四步链式处理（嵌入 → 检索 → 聚合 → 增强生成）。LLM 侧 KIMI + MIMO 双模型方案，向量存储为自研 PostgreSQL JSONB + JS 余弦相似度。系统覆盖 10 张业务表、36 个 CRUD 端点，工具输入输出均通过 Zod schema 约束。
