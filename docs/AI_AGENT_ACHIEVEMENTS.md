# 使用 Agent / AI 驱动构建的具体成果

这个项目在传统的医疗后台管理系统上，用 LangChain 搭了一套 AI 能力。底层接了两个模型：KIMI 负责推理和分析，MIMO 负责生成医学文档（还能联网搜资料）。核心是个自己写的 AgentManager，不走 LangChain 原生的 AgentExecutor，而是先让 LLM 判断用户想说啥（查数据、聊医学、搜知识库），再分发到对应的工具去干活。工具一共 8 个，最有意思的是 databaseQuery——用户用大白话问「最近一周收了多少高血压病人」，Agent 先让 LLM 把问题转成查询计划，再走 Prisma 去 10 张表里查，最后让 LLM 把结果润色成一段人话返回。还有个 RAG 链路：17 篇医学文档用阿里云的 embedding 模型向量化之后存进 PostgreSQL，用户提问时用余弦相似度搜出相关内容，塞进提示词里让 LLM 带着出处回答。患者详情页里还做了一个 AI 分析功能，把患者从头到尾的病历数据打包喂给 LLM，让它按固定格式输出风险评估、关键发现、诊疗建议这些结构化结果，分析完自动存回数据库，下次打开还能看到。前端这边为了填病历方便，把原来那些 JSON 文本框换成了标签、键值对、动态行三种可视化组件，填完自动转成正确格式存库。
